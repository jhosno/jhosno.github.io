Los formularios son componentes recurrentes en el desarrollo de aplicaciones, y su extensión y complejidad puede variar según las necesidades, algunos son cortos, con validaciones predeterminadas del navegador, y otras pueden ser muy largas, con condicionales y validaciones personalizadas bastante elaboradas. Esto puede traducirse muchas líneas de código, que después de un tiempo se vuelven engorrosas de mantener.

Por esto, es conveniente usar librerías, como Formik que te permite hacer formularios robustos con pocas líneas de código, teniendo esto en mente, vamos a explorar que es, cuáles son sus virtudes y como implementarlo.

Formik es una librería de React para manejar formularios, desarrollado por Jared Palmer, que ha dejado en la documentación oficial más información relevante que puede que quieras consultar [[link](https://formik.org/docs/overview)]. Entre herramientas que nos brinda formik están: manejar en las validaciones y los mensajes de error, manejar los states del formulario, y manejar en los envíos de los mismos. Aparte de ser una manera declarativa, intuitiva y adaptable.

## Características

- Es extremadamente flexible, te permite decidir como y cuanto utilizar la librería. Puedes agregar reglas de validación personalizadas, con Yup. Para quienes aún no están familiarizados, Yup es una librería de JS para construir, analizar y validar patrones.
- Código menos redundante.

### desventajas de Formik

- Es exclusivo para react y react native
- Los controladores de los componentes generan re-rendering

## Cómo aplicarlo

Para instalarlo basta con escribir en la consola sobre la carpeta de tu proyecto react la siguiente línea en tu proyecto de react:

```bash
npm install formik
```

Vamos a hacer un pequeño form emulando una agenda de direcciones, pero antes vamos a revisar los elementos más comunes a utilizar con Formik:

`Formik` es el label principal que va a contener el resto del los elementos de del formulario, como valores iniciales o la función onSubmit. Estos son alguno de los atributos más usados de `formik`:

- `initialvalues` es un state propio de formik para manejar el estado de los valores contenidos en los inputs

- `validationSchema` este es una propiedad que permite recibir esquemas de validación anudas a los valores iniciales.

-  `onSubmit` es la función que maneja los valores enviados por el formulario

- *`enableReinitialize`* es una función que permite reiniciar los valores del form, una vez noté que hay algún cambio, es ideal para cuando se usa el mismo componente de form para crear y editar

`Form` Es el contenedor de los elementos de formulario, pero que toma automáticamente los atributos y configuraciones definidas en el contenedor `Formik`

`Field` es el equivalente de input, incluso puedes pasarle los mismos atributos que una etiqueta input

Y la estructura

```jsx
import { Formik, Form, Field } from 'formik'

const FormClient = () => {

  return (
		<Formik
			initialValues=
			{
        name: ''
        direccion: ''
      }
	  }
			onSubmit={(values)=>{
				console.log(values)
			}}
		>
{/*para conectar con el initialValue vamos
a retornar el form a través de una función*/}
		{() => {
		        return (
		<Form>
			<div>
				<label>nombre</labe>
				<Field type="text" name="nombre"/>
			</div>
			<div>
				<label>Dirección</labe>
				<Field as="textarea" type="text" name="direccion"/>
			</div>
		</Form>
		)}
		</Formik>

);
};

export default FormClient;

```

Como verás, empezar a utilizar Formik en tus proyectos es bastante sencillo y poderoso.
